# distribution-fixe-quartz
分布式定时任务调度

``````
第一步
    获取到所有的注解信息
    
第二步
    获取到的信息， 和当前服务的名字， 端口 一起发送到 server
    
第三步
    server接收到后， 写入到数据库中(重启的时候独占， 并且清空所有表)
    
第四步
    解析得到的信息， 并且组装成服务，方法， 隐藏服务地址， 今后会根据服务和方法，分别调用
    
第五步
    在客户端，编写执行代码
整体结构
实现简单的调度


调度中心需要做好的准备：
1. 基本信息的拿去
2. 对调度的日志的拿取
3. 对调度的任务的状态的拿取
4. 对调度的报表的获取和展现
```````
